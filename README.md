# Fiverr Order Bot

Telegram-бот, который помогает **начинающему** фрилансеру находить простые заказы на Fiverr, анализировать описания и готовить честные черновики откликов для **ручной** отправки.

> ⚠️ Бот **не парсит** Fiverr, **не отправляет** сообщения за пользователя, **не обходит** капчу и **не нарушает** ToS платформы. Он работает как ассистент: даёт рекомендации, оценку и черновик отклика — финальное решение всегда за человеком.

---

## Возможности

- 🛠 **Профиль фрилансера** — языки, уровень опыта, интересующие типы задач, мин. бюджет, макс. сложность, имя для подписи, ссылка на портфолио.
- 🔍 **Готовые поисковые ссылки** для Fiverr с подборкой ключевых слов под начинающего разработчика.
- 🧪 **Анализ заказа** — пользователь вставляет описание, бот возвращает разбор: категорию, сложность, риски, нужные навыки, вопросы клиенту, рекомендуемую цену и срок.
- 📊 **Scoring 0–100** по 5 критериям: совпадение с навыками, простота, бюджет, ясность, низкий риск.
- ✉️ **Черновик отклика** — короткий, честный, без шаблонных «I am the best» и пустых обещаний.
- 🤖 **LLM-анализ опционален**: если есть OPENAI_API_KEY — бот делает осмысленный разбор; если нет — работает на rule-based heuristics.

## Архитектура

Кратко:

- `bot.py` — точка входа. Создаёт `Bot`, `Dispatcher`, регистрирует роутеры, запускает long polling.
- `config.py` — загружает `.env` и собирает dataclass `Settings`.
- `database.py` + `models.py` — async SQLAlchemy + aiosqlite, одна таблица `user_profiles`.
- `keyboards.py` — все inline и reply клавиатуры.
- `handlers/` — отдельный роутер на каждый сценарий: `start`, `profile` (FSM-опросник), `search`, `analyze`, `help`.
- `services/scoring.py` — rule-based оценка по 5 критериям из ТЗ.
- `services/llm.py` — обёртка над OpenAI-совместимым API (опциональная).
- `services/fiverr_keywords.py` — список ключевиков и генератор ссылок поиска.
- `prompts/` — текстовые промпты для анализа и генерации отклика.

## Установка и запуск

Требуется **Python 3.11+**.

```bash
# 1. Клонировать репозиторий
git clone <repo_url>
cd fiverr_order_bot

# 2. Создать виртуальное окружение
python3.11 -m venv .venv
source .venv/bin/activate          # Linux/macOS
# .venv\Scripts\activate           # Windows

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Создать .env из шаблона и заполнить значения
cp .env.example .env
# открой .env и впиши BOT_TOKEN (обязательно) и OPENAI_API_KEY (опционально)

# 5. Запустить бота
python bot.py
```

Бот стартует в режиме long polling — никаких внешних webhook не требуется.

### Минимальный `.env`

```env
BOT_TOKEN=123456789:replace_with_real_token
# Опционально — без него работает rule-based анализ
OPENAI_API_KEY=
```

### Альтернативные LLM-провайдеры

В `services/llm.py` используется `AsyncOpenAI`, поэтому подойдёт любой OpenAI-совместимый эндпоинт. Просто поменяй `OPENAI_BASE_URL` и `OPENAI_MODEL` в `.env`:

| Провайдер  | OPENAI_BASE_URL                      | Пример OPENAI_MODEL              |
| ---------- | ------------------------------------ | -------------------------------- |
| OpenAI     | `https://api.openai.com/v1`          | `gpt-4o-mini`                    |
| OpenRouter | `https://openrouter.ai/api/v1`       | `anthropic/claude-3.5-sonnet`    |
| Together   | `https://api.together.xyz/v1`        | `meta-llama/Llama-3-8b-instruct` |
| Ollama     | `http://localhost:11434/v1`          | `llama3.1`                       |

## Безопасность и ограничения

Бот **не делает**:

- автоматическую отправку сообщений на Fiverr;
- обход капчи или антибота;
- агрессивный парсинг / scraping Fiverr;
- использование чужих cookies / session tokens;
- массовые отклики;
- ничего, что нарушает [ToS Fiverr](https://www.fiverr.com/terms_of_service).

Бот **делает**:

- работает как локальный ассистент в Telegram;
- даёт рекомендации;
- генерирует черновики откликов (пользователь отправляет вручную);
- хранит только профиль фрилансера в локальном SQLite;
- не сохраняет тексты заказов.
